# Convenciones Git

## Ramas de estudiantes

```text
feature/nombre-apellido
```

## Ramas demo

```text
demo/t5-api-verde
demo/t5-api-roja
demo/t6-coverage-verde
demo/t6-coverage-roja
demo/t9-artifact
demo/t10-feature-flag
demo/t10-rollback
```

## Mensajes de commit

Usar mensajes breves y descriptivos:

```text
Agrega prueba para producto inexistente
Corrige contrato de catálogo
Configura pipeline de cobertura
```

## Pull Requests

Cada PR debe incluir:

- Qué cambia.
- Cómo se probó.
- Qué validó el pipeline.
